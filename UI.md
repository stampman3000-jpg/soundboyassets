# User Interface

## Overview

The user interface has been developed alongside the hardware rather than as
a separate layer. Every change to the software has been tested with the
physical controls in mind, with the aim of making Soundboy feel like an
instrument rather than a computer.

The interface continues to evolve through experimentation, with each
iteration helping to identify what feels intuitive during music making.

---

## Design Principles

The interface is guided by a few simple ideas:

- Minimise unnecessary menu navigation
- Keep frequently used controls within easy reach
- Provide immediate visual feedback
- Encourage experimentation rather than interruption
- Prioritise making music over configuration

---

## Display

The current interface is built around a 256×64 OLED display.

The limited screen size has encouraged a simpler approach to information,
focusing on only what is useful at any given moment rather than trying to
display everything at once.

![Current interface](assets/photos/ui-current.jpg)

---

## Physical Interaction

Soundboy is designed around dedicated physical controls.

Rather than relying on a touchscreen, interaction is based on:

- Dedicated buttons
- Rotary encoder navigation
- Immediate visual feedback
- One-handed operation where possible

The aim is to reduce the distance between an idea and the resulting sound.

## Design Decisions

Some examples of decisions made during development:

### Why an OLED?

A dedicated display keeps the interface focused and encourages interaction
through the hardware rather than external devices.

---

### Why physical buttons?

Dedicated controls allow frequently used functions to become muscle memory,
making the instrument feel more immediate.

---

### Why not a touchscreen?

Touchscreens offer flexibility, but physical controls provide tactile
feedback that is difficult to replace during performance.

---

## Challenges

Designing for a small display introduces useful constraints.

Some ongoing challenges include:

- Presenting enough information without clutter
- Reducing menu depth
- Keeping navigation predictable
- Balancing flexibility with simplicity

Each iteration helps refine both the interface and the workflow.

---

## Future Direction

Areas currently being explored include:

- Improved parameter editing
- Better visual feedback
- Faster navigation
- More consistent layouts
- Animation where it improves usability rather than decoration
